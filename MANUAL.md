# CellAppCP3 使用手册

## 快速开始

1. 将需要识别的图像放入 `input/` 文件夹中。
2. 在终端运行：`python cells_find.py`
3. 结果将在 `results/` 文件夹中生成。

---

## settings.txt 参数说明

修改 `settings.txt` 后保存，下次运行自动读取，无需改代码。

| 参数 | 默认值 | 说明 | 建议范围 |
|------|--------|------|----------|
| `diameter` | 120 | 细胞直径（像素），较小值识别更小的细胞 | 50–200 |
| `flow_threshold` | 0.95 | 流场一致性阈值，越低越宽松 | 0.1–1.0 |
| `cellprob_threshold` | 1.0 | 细胞概率阈值，越低检测越多 | -6.0–6.0 |
| `min_size` | 200 | 最小细胞面积（像素），排除过小区域 | 50–500 |
| `resample` | 0 | 是否重采样，1=开启（更慢更准） | 0 或 1 |
| `niter` | 200 | 流场迭代次数，越少越快 | 50–400 |
| `hull_comp` | 0.85 | 凸包完整度阈值，越低越宽松 | 0.6–0.95 |
| `circle_comp` | 0.65 | 圆形完整度阈值，越低越宽松 | 0.4–0.85 |
| `dark_threshold` | 15 | 暗区过滤，低于此均值的区域跳过 | 5–30 |
| `area_ratio` | 0.15 | 面积过滤，小于中位数此比例的细胞剔除 | 0.05–0.3 |
| `circularity` | 0.5 | 圆形度阈值，越低接受越不圆的形状 | 0.3–0.7 |
| `min_pixels` | 50 | 亮度计算最少像素数 | 20–100 |
| `brightness_top_pct` | 0.05 | 亮度取 top 百分比 | 0.01–0.1 |
| `top_n` | 3 | 高亮显示最亮的前 N 个细胞 | 1–20 |
| `large_mode` | 1 | 大图处理方式，1=自动分块，0=整图 | 0 或 1 |
| `tile_w` | 2048 | 分块宽度（像素） | 1024–4096 |
| `tile_h` | 1080 | 分块高度（像素） | 512–2160 |
| `tile_overlap` | 0.2 | 分块重叠比例 | 0.1–0.3 |
| `crop_pad` | 2.0 | Top 裁剪图范围倍数（相对细胞半径） | 1.5–3.0 |
| `contour_thickness` | 2 | 描边线宽（像素） | 1–5 |
| `font_scale` | 0.7 | 标注文字大小 | 0.3–1.5 |

## 输出文件说明

每张图片处理后，在 `results/{图片名}/` 目录下生成：

| 文件 | 说明 |
|------|------|
| `{图片名}_result.png` | 标注结果图，Top N 绿色高亮，其余黄色 |
| `{图片名}_raw_detections.png` | 诊断图，过滤前所有原始检测点（绿色圆点） |
| `{图片名}_top{n}.csv` | Top N 细胞的坐标和亮度值 |
| `{图片名}_data.csv` | 全部有效细胞数据（编号、直径、坐标、亮度） |
| `top1.png` / `top2.png` / ... | Top N 细胞的放大裁剪图，左上角标注排名 |

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| 细胞没被检测到 | 降低 `cellprob_threshold`（如改为 0.0 或 -2.0） |
| 检测到太多噪点 | 提高 `hull_comp` 和 `circle_comp` |
| 暗细胞被过滤 | 降低 `dark_threshold` |
| 运行太慢 | 降低 `niter`，关闭 `resample=0` |
| 大图内存不足 / 崩溃 | 确认 `large_mode=1`，减小 `tile_w` / `tile_h` |
| 裁剪图太小看不清 | 增大 `crop_pad`（如改为 3.0） |
